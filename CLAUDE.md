## 💬 소통 방식
- 기술 용어는 **쉽게 풀어서** 설명해. 초보자 기준.
- 실행 **전에** "무엇을 할 건지" 먼저 말해줘.
- 에러가 나면 **원인과 해결책**을 함께 알려줘.
- 모르는 건 **"모르겠어요"** 라고 솔직히 말해.

## 🛠️ 작업 원칙
- 기존 파일 수정보다 **새 파일 제안**을 우선해.
- 변경사항은 **"1. xxx, 2. yyy"** 요약으로 알려줘.
- 큰 작업은 **체크리스트**로 나눠서 보여줘.

---

## 🎯 작업 방식 (6가지 원칙)
### 1. 큰 작업은 계획부터: `tasks/todo.md`에 계획부터 적고 컨펌받기.
### 2. "다 했다" 선언 전 동작 확인: 브라우저에서 결과 확인 후 완료 선언.
### 3. 에러 나면 멈추기: 근본 원인부터 수정.
### 4. 한 번에 조금씩: 한 단계씩 바꾸고 확인하기.
### 5. 모르면 "모르겠어요": 솔직하게 질문하기.
### 6. 작업 기록 남기기: `tasks/todo.md`, `tasks/progress.md` 활용.
