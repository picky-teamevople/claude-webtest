# 🔒 SECURITY.md — 비상 매뉴얼

> Claude Code가 이 프로젝트에서 지켜야 할 보안 규칙과, 문제가 생겼을 때 대응하는 방법을 정리한 문서입니다.
> 원본 규칙은 `CLAUDE.md`에 있으며, 이 문서는 그 규칙을 실제 상황에 어떻게 적용할지 풀어서 설명합니다.

---

## 1. 지켜야 할 기본 규칙

| 규칙 | 설명 |
|---|---|
| API 키·비밀번호 마스킹 | 화면에 보여줄 때는 `sk-ab12****` 처럼 일부만 보이게 가립니다. |
| `.env`, `*.key` 커밋 금지 | 이런 파일은 절대 `git add`, `git commit` 하지 않습니다. |
| 하드코딩 금지 | 코드에 직접 키 값을 적지 않고, 항상 환경변수(`process.env.XXX` 등)로 참조합니다. |

---

## 2. 만약 실수로 키가 노출됐다면 (사고 대응 절차)

**초보자 기준으로, 아래 순서대로 진행하면 됩니다.**

1. **당황하지 말고 멈추기** — 추가로 커밋/푸시하지 않습니다.
2. **키 무효화(rotate)** — 노출된 키를 발급한 서비스(예: OpenAI, AWS, GitHub 등)에 가서 해당 키를 즉시 삭제하고 새 키를 발급합니다. 이 단계가 가장 중요합니다. 코드에서 지워도 키 자체가 여전히 살아있으면 위험합니다.
3. **git 기록에서 제거 여부 판단**
   - 아직 원격(GitHub 등)에 푸시하지 않았다면: 커밋을 되돌리는 것이 비교적 안전합니다.
   - 이미 푸시했다면: git 기록에서 완전히 지우는 작업(`git filter-repo` 등)은 **위험한 작업**이므로, 반드시 사용자에게 먼저 물어보고 진행합니다.
4. **환경변수로 교체** — 새 키는 `.env` 파일에 넣고, 코드에는 `process.env.KEY_NAME` 형태로만 참조하도록 수정합니다.
5. **`.gitignore` 확인** — `.env`, `*.key`가 `.gitignore`에 등록되어 있는지 확인합니다. 없다면 추가합니다.
6. **기록 남기기** — `tasks/progress.md`에 무슨 일이 있었고 어떻게 해결했는지 기록합니다.

## 3. 위험한 작업 — 실행 전 반드시 확인

아래 작업은 Claude가 **먼저 사용자에게 물어본 뒤에만** 실행합니다.

- 파일·폴더 삭제 (`rm -rf`, `Remove-Item -Recurse -Force`)
- Git 되돌리기 (`git reset --hard`)
- 강제 푸시 (`git push --force`)
- git 기록 재작성 (`filter-repo`, `rebase -i` 등)
